Setting up Petal

The Quick Way

Include the link code inside the <head> of your HTML file. Place it before any other site page stylesheets.

<link rel="stylesheet" type="text/css" href="https://shakr-petal.s3-ap-northeast-1.amazonaws.com/edge/petal.min.css">

Now you can use all the Petal classes and styles in your project.


The Custom Way

You can customize some settings like colors and font options to fit the look of your project design and manually compile your own build of Petal.

If you already have a LESS compiler set up in your project, just copy the files in the less folder from the Petal repository to your local project folder. It's recommended you put the Petal files in a separate directory (presumably, named "Petal") to avoid confusion.

Alternatively, you can also clone the whole repository and use grunt to compile Petal, then add the compiled CSS file to your project.

To customize options, change the values of variables in petal.less before compiling.

Layout

Base structure

Set a container with .container class in which you will place .rows. Use appropriate HTML5 semantic tags such as <section>,<article>,<header> and <footer> along with the class.

You must have .row sub-containers for child elements to align to the page width.

Container widths and responsive layout

Container width is set as a LESS variable @container-width in the base petal.less file. Default value is 980px; modify the value to change fixed width.

Responsive layout is enabled by default, and the page will automatically fit the window width when screen width is smaller than the container width.

Options
  • .hollow : Hollow style alternative
  • .disabled OR :disabled (attribute) : Disabled state
  • .ondark on button OR .dark on parent : Alternative style on dark backgrounds
  • .whitetext : Force white text on buttons

Including additional colors

You can add new button colors easily. First add your color as a variable in color-palette.less then add a line in buttons.less.

// Add colors here
&.my-color    { .generate-btn('my-color') }
&.my-color2   { .generate-btn('my-color2'; 'custom') }

The 'custom' option is provided for when you want to specify the l1, l2, d1, d2 shades manually. Define all 5 variables including base color in color-palette.less and append the 'custom' option at the end.

Checkboxes and Radioboxes

<!-- Checkbox -->
<input class="checkbox" type="checkbox" id="demoCheck1" checked disabled>
<label for="demoCheck1">Checkbox 1</label>

<!-- Radiobox -->
<input class="radiobox" type="radio" name="demoRadio" id="demoRadio1" checked>
<label for="demoRadio1">Radiobox 1</label>

Custom checkbox and radiobox styles uses CSS3 :after and :before pseudo elements to override default browser appearance.

Therefore, <label>s are a requirement and must be placed after the <input>s. Also you need to append .checkbox or .radiobox classes to each <input>s for correct appearance.

.dark classes on parent elements are supported for darker check/radiobox style variations.


Customizable options

@radiobox-border-radius: 100%;
@checkbox-color: @primary-accent-color;
@checkbox-disable-animation: false;

Selectboxes

Selectboxes uses Selecter jQuery Plugin by Ben Plum (MIT License). The javascript file is included as-is in petal-functions.js, but the CSS code is completely customized to fit the Petal style.

The Selecter plugin is not activated by default because of possible usability issues. To initialize, include the line of code below in your site page (change the selector to whichever selectbox you want to apply to).

$(".myclass1 select.selectbox").selecter();

Then append the .selectbox class to properly apply styles.

Refer to the Selecter documentation page for more options.


Customizable options
// selectbox
@selectbox-border-width: @input-border-width;
@selectbox-color: @primary-accent-color;
