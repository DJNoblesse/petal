Layout

Base structure

Set a container with .container class in which you will place .rows. Use appropriate HTML5 semantic tags such as <section>,<article>,<header> and <footer> along with the class.

You must have .row sub-containers for child elements to align to the page width.

Container widths and responsive layout

Container width is set as a LESS variable @container-width in the base petal.less file. Default value is 980px; modify the value to change fixed width.

Responsive layout is enabled by default, and the page will automatically fit the window width when screen width is smaller than the container width.

All button color sets are generated dynamically when the LESS file is compiled. To add a new button color you can use, just write another line in buttons.less file where it calls either .generate-btn() or .generate-btn-custom() LESS mixin functions and compile (build) Petal.

The difference between the first 'static' one is that it requires only one base color (it will generate all other needed shades automatically) and the 'custom' one requires all 5 shade variants including base colors variables set in the color-palette.less file. Use the custom generator mixin if you want more finer control over the tones of your color set.
